# Implementation Notes

This folder is for non-normative implementation notes, field diagnostics, and issue reports collected from real sessions.

These documents do **not** define protocol or policy by themselves.

Use this folder when:

- a real session exposes a runtime or UX problem
- an implementation diagnosis is useful for later repair
- a field report is needed before a formal protocol change is designed

Do **not** treat documents here as canonical protocol overrides.

If a note here leads to a real protocol or workflow change, that change must be written separately into the appropriate canonical surface, for example:

- `docs/protocols/`
- `docs/AITP_*.md`
- `research/knowledge-hub/*.md`

Recommended naming:

- `YYYY-MM-DD-<short-issue-name>.md`

Recommended content:

- session context
- observed behavior
- evidence paths
- impact
- likely root cause
- proposed fixes
