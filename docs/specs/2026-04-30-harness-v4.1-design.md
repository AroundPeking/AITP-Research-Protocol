# AITP Harness v4.1 — Grounded Modification Plan

## 0. What We Now Know (That The Earlier Spec Got Wrong)

From the deep code reading:

| Earlier assumption | Actual code reality |
|---|---|
| `_build_execution_brief` exists | It does NOT exist. Domain injection must be added to gate return dict itself. |
| Gate checks could be "replaced" with content checks | Gate uses `h not in body` substring matching. Content checks must be ADDED as additional checks, not replace heading checks. |
| AITP hooks are similar to Superpowers | AITP hooks do skill LOOKUP and print instructions. Superpowers hooks do full skill CONTENT INJECTION via JSON-escaped `additionalContext`. Fundamentally different. |
| "Just add Red Flags table to using-aitp.md" | Red Flags pattern requires adversarial testing baseline. First version must come from actual violations observed in this QSGW topic. |

## 1. Architecture: What Changes Where

```
Superpowers pattern               →  AITP implementation
─────────────────────────────────────────────────────────

  content as additionalContext       using-aitp content (not just print filename)

  (12 entries from real violations)    (entries from this QSGW topic's violations)

                                     single-question refinement

                                     (injected into agent's context)
Red Flags accumulated per-session →  runtime/red_flags_learned.md accumulates
```

## 2. File-by-File Changes

### 2.1 `hooks/session_start.py` — Rewrite to Inject Skill Content

**Current behavior** (from source):
- Reads topic stage from `state.md`
- Calls `evaluate_l*_stage()` to get skill name
- Prints: `"AITP: MANDATORY -- read and follow skills/<skill>.md before continuing."`
- Prints tool catalog

**Problem**: Agent sees a filename to read, not the actual skill content. It can ignore the instruction.

**Change**: Inject the FULL content of `using-aitp.md` as `additionalContext`, exactly like Superpowers does.

```python
# New function in session_start.py
def _build_aitp_injection(topics_root, active_slug):
    """Build the JSON context injection, Superpowers-style."""
    # Read using-aitp skill content
    skill_path = Path(__file__).parent.parent / "deploy" / "templates" / "claude-code" / "using-aitp.md"
    skill_content = skill_path.read_text(encoding="utf-8")
    
    # Get active topic context
    state_fm, _ = _parse_md(Path(topics_root) / active_slug / "state.md")
    stage = state_fm.get("stage", "L0")
    posture = state_fm.get("posture", "discover")
    gate = state_fm.get("gate_status", "unknown")
    
    # Build the injection
    context = f"""<EXTREMELY_IMPORTANT>
{skill_content}
</EXTREMELY_IMPORTANT>

AITP Runtime State:
- Topics root: {topics_root}
- Active topic: {active_slug}
- Stage: {stage}
- Posture: {posture}
- Gate status: {gate}
"""
    return context
```

The hook output format matches Superpowers: `{"hookSpecificOutput": {"hookEventName": "SessionStart", "additionalContext": "..."}}` for Claude Code, `{"additionalContext": "..."}` for others.

Also update `hooks/compact.py` to re-inject the same context after compaction.

### 2.3 `skills/skill-discover.md` — Checklist-Driven Discovery

**Current**: "Search systematically... Register each source... Fill registry..."
**Problem**: Agent determines what to register and in what order, without user dialogue.

**Change**: Replace the workflow with a checklist pattern:

```markdown
## Checklist
You MUST create a task for each item and complete them in order:

1. **Check L2 first** — `aitp_query_l2_index(topics_root)`. Report what exists.
2. **Ask: what is the single most important source?** — ONE question via AskUserQuestion.
```

Remove: "Search systematically using available tools" — this encourages batch registration without dialogue.

### 2.6 `brain/state_model.py` — Gate Content Checks + Domain Injection

**Current gate logic** (from code): `_missing_required_headings` checks `h not in body` (substring match). `_missing_frontmatter_keys` checks `str(fm.get(key, "")).strip()` non-empty.

**Changes**:

**A. Add `_check_heading_content(body, heading, min_chars=80)`**:
```python
def _check_heading_content(body: str, heading: str, min_chars: int = 80) -> bool:
    """Check that a Markdown heading has substantive body content."""
    idx = body.find(heading)
    if idx == -1:
        return False
    # Find content between this heading and next ## or end
    content_start = idx + len(heading)
    remaining = body[content_start:]
    next_heading = remaining.find("\n## ")
    if next_heading == -1:
        section_body = remaining
    else:
        section_body = remaining[:next_heading]
    # Strip whitespace, heading markers, and empty lines
    cleaned = section_body.strip()
    return len(cleaned) >= min_chars
```

Apply to: `## Overall Verdict` (L0, min 200 chars), `## Coverage Summary` (L1, min 100 chars).

**B. Add `_check_derivation_steps(topics_root)`**:
```python
def _check_derivation_steps(topics_root: str) -> tuple[int, list[str]]:
    """Count derivation steps and return step IDs."""
    steps_dir = Path(topics_root) / "L2" / "graph" / "steps"
    if not steps_dir.exists():
        return 0, []
    step_files = list(steps_dir.glob("*.md"))
    step_ids = [f.stem for f in step_files]
    return len(step_files), step_ids
```

Used in `evaluate_l3_stage`: requires `step_count >= 1` for `derive` activity, `step_count >= 0` (can be empty initially) for other activities. This gates L3→L4 on actual derivation work being recorded.
