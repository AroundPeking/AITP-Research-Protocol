# Adversarial Collaborator Design: Skills as Socratic Interlocutors, Not Checklist Generators

Design direction: (1) Cut Python to ~15 tools - pure storage+search only. (2) Python never judges content quality. (3) Each skill file gives the agent a role and a set of adversarial questions, not a form to fill. (4) L2 is conversationally curated, not automatically populated. (5) Retrieval is where Python earns its keep - semantic search, LaTeX indexing, regime filtering.
