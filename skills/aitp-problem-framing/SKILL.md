# AITP Problem Framing

## Environment gate (mandatory first step)

- Confirm the task is already inside an AITP-governed workspace or topic.
- Read the current `research_question.contract.json`, `idea_packet.json`, and the main L1 source-intake rows before rewriting the problem statement.

## When to use

- A topic question sounds procedural rather than physical.
- The report needs a stronger opening section.
- The current topic lacks a clear statement of motivation, literature position, or non-goals.

## Workflow

1. State the physical question in one sentence.
2. State why the question matters physically.
3. State what sources or prior results the topic leans on.
4. State the exact bounded scope of the current notebook.
5. State what is explicitly out of scope.

## Hard rules

- Do not replace a real physical question with runtime jargon.
- Do not claim broad novelty if the current work only restores or clarifies a bounded route.
- Always separate `what this notebook establishes` from `what remains outside scope`.
