# AITP L3-L4 Round

## Environment gate (mandatory first step)

- Confirm the topic already has an active run id.
- Read the current iteration packet if one already exists before starting a new round summary.

## When to use

- A topic needs to record one bounded research round.
- The notebook should show plan-test-result flow instead of only event logs.
- A returned L4 result needs a clear L3 synthesis.

## Workflow

For each round, write:

1. `Question for this round`
2. `Physical hypothesis or route`
3. `Plan / test criteria`
4. `Returned check or benchmark result`
5. `What changed in our understanding`
6. `Decision for the next round`

## Hard rules

- Do not write a round as if the result was already known at the start.
- Separate the planned test from the returned evidence.
- The synthesis must say what changed, not only what was done.
